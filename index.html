Get started

    API Endpoint

        https://api.endpoint.com/

The Webtech API provides programmatic access to read Favourite content data. Retrieve our favourite content, provide your own favourite conten, update the content, filter them, etc.

To use this API, you need an API key. Please contact us at s.j2.brouwer@student.vu.nl to get your own API key.

get favourite content


# Here is a curl example
curl \
-X POST http://api.webtech.com/favourites/get \
-F 'secret_key=your_api_key' \
-F 'offset=0' \
-F 'limit=50'

To get the favourite content you need to make a POST call to the following url :
http://api.webtech.com/favourites/get

QUERY PARAMETERS

Field Type Description
secret_key String Your API key.
search String (optional) A search word to find content by name.
year Integer (optional) a integer to filter content by year
limit Integer (optional - default: 10) A limit on the number of objects to be returned, between 1 and 100.

Errors

The Webtech API uses the following error codes:

Error Code Meaning
X000 Some parameters are missing. This error appears when you don't pass every mandatory parameters.
X001 Unknown or unvalid secret_key. This error appears if you use an unknow API key or if your API key expired.
X002 Unvalid secret_key for this domain. This error appears if you use an API key non specified for your domain. Developper or Universal API keys doesn't have domain checker.
X003 Unknown or unvalid user token. This error appears if you use an unknow user token or if the user token expired.
